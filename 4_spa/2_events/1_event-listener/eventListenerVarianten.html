Verschiedene Möglichkeiten, Event-Listener hinzuzufügen

Möglichkeit 1: Direkt in HTML als Attribut. Ihr sollt diese Variante NIE benutzen!

Lorem ipsum dolor sit amet consectetur adipisicing elit. Saepe ipsa consequatur ratione iure accusantium quidem sed dolores, animi quisquam. Eius aspernatur ex voluptatibus accusamus labore, ipsam illo modi dicta ullam.

Möglichkeit 2: Als DOM-Elementeigenschaft hinzufügen. Diese Variante ist ok, aber ihr sollt sie trotzdem nicht benutzen, weil die dritte Variante besser ist

Lorem ipsum dolor sit amet, consectetur adipisicing elit. Eum, quo? Nobis fugiat, sit excepturi a expedita quia temporibus, vel rem quae pariatur libero neque, voluptas animi ipsum aperiam quo? Unde!

Möglichkeit 3: Mit element.addEventListener. Das hat mehrere Vorteile:

  • Man kann einem Element zum selben Event mehrere Listeners hinzufügen. Das werdet ihr bei euren Übungen und kleineren Projekten eher nicht machen. Aber bei größeren Projekten kann es praktisch sein
  • element.addEventListener kann ein optionales drittes Argument nehmen, mit dem man 3 Sachen einstellen kann. Eine davon ist { once: true }. Damit wird der Event Listener nur einmal aufgerufen und danach entfernt (man müsste dann nicht selber mit removeEventListener arbeiten)
  • Man kann mit benutzerdefinierten Events arbeiten (eine fortgeschrittene Programmiertechnik, kein Teil des Lehrplans)
