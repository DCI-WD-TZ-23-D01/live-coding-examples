Was ist eine Single Page Application

Single Page applications sind einzelne Seiten in komponenten, die viele informationen liefern können, und spezifische inhalte aktualisieren, ohne die seite zu wechseln oder neu laden zu müssen.

Wie wir an dem beispiel von Youtube gesehen haben, sehen wir das sich außer des hauptinhaltes nichts ändert, das menu bleibt gleich, und die seite wird nicht neu geladen. Also werden hier nur spezifische inhalte aktualisiert.

SPA Architektur

Die SPA ist eine webanwendung, oder "webseite", die mit dem benutzer interagiert, indem sie aktuelle inhalte dynamisch umschreibt, anstall neue seiten vom server zu laden.

Dieser ansatz minimiert die unterbrechung der benutzererfahrung zwischen den aufeinanderfolgenden seiten, so das die anwendung sich mehr wie eine desktopanwendung also ein "computerprogramm" im klassischen sinne verhält, weswegen es logisch ist, und WEB-ENTWICKLER und nicht WEB-DESIGNER zu nennen.

Auf den meisten webseiten gibt es viele, sich wiederholende inhalte, einige davon bleiben immer gleich, egal wo der benutzer sich hinbewegt, zum beispiel der header, der footer, die navigationsleiste, und so weiter, allgemein bleiben viele elemente im layout gleich.

SPAs machen sich die wiederholung von elementen, songenannten komponenten zu nutze, indem man diese anlegt und immer wiederverwenden kann.

Bei traditionellen multi-page applications würden wir bei jeder seite die wir anfragen, immer die ganze seite mit allen inhalten geliefert bekommen.

Das konzept MVC

MVC steht für Model-View-Controller, eine spezielle struktur für web applikationen, in der die applikation in 3 unterschiedliche elemente aufgeteilt wird, die voneinander getrennt entwickelt werden können.

  • Controller: Die steuerung der applikation, hier sitzt die programmlogik, die die daten und die sichtbare webseite verbindet
  • Model: die datenanbindung, die mit dem controller verbunden ist, um daten auszutauschen
  • View: die darstellung der daten auf der html seite, die über den controller zum benutzer kommt.

Der vorteil dieser flexiblen art der programmierung, liegt in der einfachen wiederholbarkeit und wartung. Im team ist es auch einfacher an MVC's zu arbeiten, wenn jeder seinen bestimmten aufgabenbereich hat. Allerdings ist das setup um eine MVC aufzubauen komplizierter als eine normale HTML webseite, und man ist abhängig von modulen anderer entwickler.

Was ist React?

React it eine JavaScript framework bibliothek von Facebook (Meta). Genau wie bootstrap von X (Twitter) kommt, hat facebook also mit react einen großen einfluss auf das gehabt, was wir als modernes web kennen.

React wurde entwickelt um interaktive benutzeroberflächen zu erstellen. Der hauptvorteil von React ist, dass man sehr einfach verschiedene views oder einzelne kleine teile, genannt komponenten, erstellen kann, die auf anfrage einfach mit neuen daten geupdated werden können. Jeder komponent ist in der lage, seine eigenen daten und strukturen zu managen, weswegen man sie fantastisch zu komplexen benutzeroberflächen zusammenfügen kann.

Da die applikationen aus einzelnen komponenten bestehen, kan man sie vorneinander getrennt updaten oder warten, ohne die gesuchte stelle auf tausenden zeilen HTML oder JavaScript suchen zu müssen.

React hat eine sogenannte state und prop verwaltung, das sind 2 wichtige konzepte die wir lernen müssen:

  • Props oder properties sind eigenschaften, die von außen an komponenten übergeben werden um die funktionalität zu erweitern, diese properties können innerhalb des komponenten verwendet werden. Das ist vergleichbar mit parametern, die wir an eine funktion übergeben.
  • States sind eigentschaften die innerhalb eines komponenten sind, um einfluss auf den zustand des views haben zu können. Diese states müssen auf spezielle art und weise gesetzt, genutzt und verändert werden, damit sie in echtzeit dynamisch verwendet werden können. DIes ist auch der grund, wieso react, react heißt, denn wenn sich ein state innerhalb einer komponente verändert, wird automatich die renderfunktion aufgerufen, was dazu führt, das die neuen daten direkt angezeigt werden, es wird also auf einen "state change" ... reagiert, "reacted

React arbeitet außerdem mit sogenannten Hooks, dies sind die states der verschiedenen lebensphasen eines komponenten, wie zum beispiel der zeitpunkt, des ladens des komponenten, den wir damit direkt managen könnne, außerdem gibt es hooks, die wir direkt einspeisen können, um dynamisch den view zu verändern.

React-Webseite

Fangen wir an mit react.

Die einfachste und schnellste möglichkeit, eine react applikation zu erstellen, ist die verwendung des tools Vite. Vite ist ein modernes build-tool, das extrem schnell ist und vor allem für kleinere bis mittelgroße projekte sehr gut geeignet ist. Es bietet eine bessere performance und entwicklungsumgebung als "create-react-app" (CRA), das als vorgänger genutzt wurde, um react applikationen zu erstellen.

Vite-Webseite

Um eine neue React-applikation mit vite zu erstellen, navigieren wir in den gewünschten ordner und gebenb den folgenden befehl ein npm create vite@latest

Wir sollten darauf achten projektnamen klein zu schreiben, und keine leerzeichen zu verwenden.

Vite wird uns dann nach einigen informatione fragen, hier wählen wir das gewünschte framework aus (in unserem fall react) und die programmiersprache (JavaScript)

Wollen wir den prozess vereinfachen, können wir den namen des projektes, sowie den framework auch direkt im CLI angeben. Dazu schreiben wir npm create vite@latest app-name -- --template react

Nachdem die grundstrutkur erstellt wurde, wechseln wir in den neu erstellten ordner, und installieren die benötigten node module, mit dem befehl npm install.

Sobald die installation abgeschlossen ist, starten wir den entwicklerserver mit npm run dev. Vite startet einen schnellen live server, der die anwendung sofort im browser anzeigt.

Jede änderung, die wir an unserer applikation vornehmen (zb. in der App.jsx datei), wird dank vite's Hot Module Reload (HMR) sofort im browser reflektiert, ohne das die seite neu geladen werden muss.

Struktur des Projektes

Vite erstellt eine ordner struktur, die minimal und optimiert ist. Die wihtigstens dateien befinden sich im /src odner:

  • main.jsx: Dies ist der einstiegspunkt der applikation, hier wird react in die HTML datei per DOM eingebunden
  • App.jsx: hier befindet sich unsere hauptkomponente, die die basis der Applikation bildet.
  • index.css: Enthält das grundelegende styling

Der Ordner public enthält statische dateien wie favicons oder andere bilder.
